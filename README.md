## 🤝🏼 협업 방식
### Commit message convention
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 형식, 정렬, 주석 수정
- refactor: 코드 리팩토링
- chore: 그 외 자잘한 변경

### Git branch strategy
- **develop - feature**
- develop : 배포 및 개발된 기능(feature)을 통합하는 브랜치
- feat/[name] : 담당자별 기능 개발을 진행하는 브랜치

### etc.
- 주 1회(토요일 14시) 회의 진행
- 깃허브 칸반 보드 활용
